APP测试总结
2018-12-03 15:48:52 # APP测试 # APP测试 # 测试理论

APP测试

1、用户体验

主要是易用性和UI测试

交互规范、友好

界面展示如颜色、文字、大小、横屏竖屏

2、功能测试

是否符合业务功能逻辑

3、专项测试

流量测试

模拟用户使用场景，找对应工具来测试

耗电量

模拟用户使用场景，找对应工具来测试

性能测试

启动速度：ddms工具查看、adb shell am start -W -n 包名/类名；冷启动&热启动

CPU占用

内存占用、是否溢出：引用的对象无法被GC

FPS帧率

页面加载响应时长

文件IO读写

APP体积大小

稳定性测试

使用monkey进行稳定测试

4、兼容性

可使用云测平台

重点关注设备型号；系统版本；屏幕尺寸；分辨率等

5、安全测试

cookie和session过期检查

抓包，日志里有无敏感信息

数据权限校验

多处登录

APP签名

反编译

认证授权

密码多次错误；是否明文存储

6、crash崩溃

可通过日志系统监控APP crash概率，控制APP 崩溃概率

SDK升级不兼容

系统版本兼容

缓存问题

内存问题

APP权限未开放

7、单元测试

建议review开发单元测试；或与开发pair共同维护单元测试

8、安装卸载升级测试

安装：安装成功过、升级、降级安装后是否能用、安装位置是否正确

中途取消安装、安装进度检查

卸载测试：成功卸载，删除对应数据

升级测试：跨多个版本

9、异常中断测试

相关技术

webview

webview是一个基于webkit引擎、展示页面的控件。

WebView控件功能强大，除了具有一般View的属性和设置外，还可以对url请求、页面加载、渲染、页面交互进行强大的处理。

  • 显示和渲染Web页面
  • 直接使用html文件（网络或本地assets中）作布局
  • 可和JavaScript交互调用

如果只是mWebView.loadUrl()加载的话，那么点击页面中的链接会在手机的默认的浏览器上打开。

如果想让页面在App内打开的话，那么得设置setWebViewClient

让WebView调用js方法

WebSettings webSettings = mWebView.getSettings(); //设置为可调用js方法 webSettings.setJavaScriptEnabled(true);

什么是JSBridge

顾名思义，JSBridge就是定义Native和JS的通信,Native只通过一个固定的桥对象调用JS,JS也只通过固定的桥对象调用Native,基本原理是:

H5->通过某种方式触发一个url->Native捕获到url,进行分析->原生做处理->Native调用H5的JSBridge对象传递回调。

ADB

常用命令

adb devices：获取设备列表及状态

adb install 路径\xx.apk：安装应用

adb install -r ：重新安装

adb uninstall <包名>：卸载APP

adb pull sdcard/pull.txt d:\ ： 复制 Sdcard 下的 pull.txt 文件到 D 盘

推送 D 盘下的 ITester.txt 至 Sdcard：

adb push d:\ITester.txt sdcard/ ： 推送 D 盘下的 ITester.txt 至 Sdcard

adb kill-server /adb start-server ：结束 adb 服务/启动 adb 服务，通常两个命令一起用，设备状态异常时使用 kill-server，运行 start-server 进行重启服务。

adb logcat：打印系统日志

APP指标分析

常规数据指标的监测

  • 用户量
  • 新用户量
  • UGC量（原创内容量）
  • 销量
  • 付费量

渠道分析

对于App来说，一般会花资源去引流量，去别的渠道拉用户。

这个时候需要监测各个渠道的好坏，哪个渠道效果好且便宜。

监测不同渠道用户的后续表现，给每个渠道的用户打分。

监测iPhone和Android用户质量，甚至不同机型的用户。

用户的核心转化率

这个依据App的核心功能，游戏里可能是付费率，电商里是购买率。

用户使用时长的监测

用户上线后的时长和预计时长对比

用户流失情况

新用户进来后，第一、三、七、三十天还在使用产品的有多少人。

活跃用户动态

关注并倾听活跃用户。

用户特征建模

将用户的各个指标特征进行描述，越详细越好。如性别，年龄，地域，手机型号，网络型号，职业收入，兴趣爱好等等
